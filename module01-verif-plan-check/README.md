# Module 01: Coverage / plan checklist

**Kind:** `lab` · Primary lab: `verif-plan-check` · **Shipped**

[← Welcome to verification planning](../module00-intro/README.md) · [Course README](../README.md) · [Test taxonomy →](../module02-test-taxonomy/README.md)

## Outcomes

After this module you can explain and practice the ideas taught by **`verif-plan-check`**, in the browser and/or with a written plan sketch.

## Two tracks (pick one or both)

### Track A — Planning docs (hands-on)

1. Open [EXAMPLES.md](EXAMPLES.md) and work the prompts (paper / markdown tables).
2. Optional templates: [`../verification_planning_management/`](../verification_planning_management/).
3. Complete [CHECKLIST.md](CHECKLIST.md).
4. Optional: `./scripts/module.sh 01 --check`.

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/verif-plan-check/index.html](http://127.0.0.1:8080/tools/verif-plan-check/index.html)
2. Live: [https://universal-verification-methodology.github.io/learning/tools/verif-plan-check/](https://universal-verification-methodology.github.io/learning/tools/verif-plan-check/)
3. Load the **starter example**, then work challenges.
4. Check off the Track B items in [CHECKLIST.md](CHECKLIST.md).

> Concept labs are literacy tools — they do not replace a real coverage DB or farm.
